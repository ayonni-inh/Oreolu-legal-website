## Client ID format
`OGA-YYYY-NNNNN` — generated by `generateClientId()` in server.ts using a module-level `_clientIdSeq` counter (initialized randomly to avoid collisions on restart). Example: `OGA-2026-00421`.

## Registration rules
- Self-registration via `/api/auth/register` is ALWAYS `role: 'Client'` — no appRole selector shown to users
- Accepts: firstName, lastName, email, phone, password, companyName, industry, position
- Sends welcome email server-side via `sendEmail()` helper (not via frontend emailService)
- Returns: clientId in OGA format, immediately ACTIVE status

## Email helper
`sendEmail(to, subject, html)` in server.ts — graceful no-op if `RESEND_API_KEY` is absent or starts with `re_123456789`. Uses `onboarding@resend.dev` as from address unless `RESEND_FROM_EMAIL` env is set.

**Why:** Resend free tier only allows sending to the account owner's email unless a domain is verified. The helper must not throw or break flows when email is unconfigured.

## Invitation roles (AdminAIPanel → /api/staff)
Both Staff AND Client invitations now go through `/api/staff` with `role: 'Client'` or `role: 'Staff'`. The endpoint generates an invite token and sends a set-password email with the appropriate copy for each role type.
